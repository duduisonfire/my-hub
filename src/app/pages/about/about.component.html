Rune Maker

This project is definitely my favorite. Rune Maker is a open-source desktop application, built with web technologies to be an assistant to the game League of Legends. And what is its real use? Before starting a match, players need to select the best combination of runes for their character, this is a lot of work, let Rune Maker do it for you.

How does the program work? Let's start by talking about how the program communicates with the League of Legends Client. Whenever the game is opened on your machine, it automatically launches a Rest API on localhost, from this API you can have full control of the game's Client. Now coming back to Rune Maker, it is conceptually very simple, it was built using Electron, React and TypeScript. Electron works with two layers, one that accesses the operating system and another that renders the screen. React is responsible for the screen rendering layer, it also does all the hard work of accessing the LoL API and my own API to get the best sets of runes for each champion (we'll talk about it later). Electron is responsible for an operation, finding the game folder that the user selected, checking if the game is open and collecting the information necessary to authenticate with the Client API. With all that in hand, the rest of the work is manipulating data and creating screens in React, nothing too complex. Ah, yes, I almost forgot, this project was created entirely for Linux (maybe in the future we will launch a version for Windows).

Rune Scraping API

This project is a complement to Rune Maker, as I said a little above, in each game the player needs to select the most efficient set of runes for their character. Naturally, Rune Maker doesn't know how to choose the best runes, that's why it asks for this API.

How does the program work? The API works in a very simple way, it basically has a CronJob configured so that, every 36 hours, the program does a Scraping on the u.gg page (a website that collects and compiles statistics about the game). It parses this information into a form that Rune Maker can receive and interpret and saves all of this in a MongoDB database. When Rune Maker needs to know about a character's rune, it sends an HTTP request with the following model: url/api/ugg/diana/mid (in this example, the chosen champion was Diana playing in mid lane), and then the API returns everything Rune Maker needs.
